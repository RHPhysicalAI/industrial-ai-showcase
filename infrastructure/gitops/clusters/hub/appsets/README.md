# Hub Cluster ApplicationSets

This directory contains ApplicationSets that manage applications on the hub cluster.

## Auto-Sync Configuration

**Status**: ✅ **ENABLED** (as of commit e3ec416)

All workload applications now have automatic sync enabled:

```yaml
syncPolicy:
  automated:
    prune: true      # Removes resources deleted from Git
    selfHeal: true   # Reverts manual changes back to Git state
```

### How Auto-Sync Works

1. **Polling Interval**: Argo CD polls the Git repository every **3 minutes**
2. **Detection**: When a new commit is detected on the tracked branch, Argo triggers sync
3. **Automatic Sync**: Applications update without manual `oc patch` or UI intervention
4. **Prune**: Resources removed from Git are automatically deleted from cluster
5. **Self-Heal**: Manual changes to cluster resources are reverted to Git state

### Branch Configuration

**Production**: ApplicationSets watch the `main` branch

**Development**: During phase3 development, PRs merge to `phase3` branch. To test auto-sync:

```bash
# Option 1: Merge PR to main (production workflow)
gh pr create --base main ...

# Option 2: Temporarily switch app to phase3
oc patch application workloads-factory-b -n openshift-gitops --type merge \
  -p '{"spec": {"source": {"targetRevision": "phase3"}}}'
```

**Note**: The ApplicationSet will override manual targetRevision changes back to `main` on next reconciliation. For phase3 testing, consider merging test PRs to `main` or use manual sync.

## End-to-End Workflow

### HIL Approval → Auto-Sync Flow

```
1. Operator approves HIL request in Console
     ↓
2. Agent creates PR to infrastructure/gitops/apps/workloads/factory-b/
     ↓
3. PR auto-merges (within seconds)
     ↓
4. Commit pushed to 'main' branch
     ↓
5. Argo CD detects change (within 3 minutes)
     ↓
6. Application auto-syncs ✅
     ↓
7. InferenceService updates (within seconds)
     ↓
8. Total time: approval → cluster updated = ~3-5 minutes
```

## Troubleshooting

### Auto-sync not triggering

1. **Check sync policy**:
   ```bash
   oc get application <name> -n openshift-gitops -o yaml | grep -A 5 syncPolicy
   ```
   Should show `automated: {prune: true, selfHeal: true}`

2. **Check branch**:
   ```bash
   oc get application <name> -n openshift-gitops \
     -o jsonpath='{.spec.source.targetRevision}'
   ```
   Should match the branch you merged PR to

3. **Force refresh** (triggers immediate poll):
   ```bash
   oc patch application <name> -n openshift-gitops --type merge \
     -p '{"metadata": {"annotations": {"argocd.argoproj.io/refresh": "hard"}}}'
   ```

4. **Check Argo CD logs**:
   ```bash
   oc logs -n openshift-gitops -l app.kubernetes.io/name=argocd-application-controller
   ```

### Manual sync (if needed)

```bash
oc patch application <name> -n openshift-gitops --type merge \
  -p '{"operation": {"initiatedBy": {"username": "admin"}, "sync": {}}}'
```
