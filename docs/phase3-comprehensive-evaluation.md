---

### 🟡 HIGH-PRIORITY GAPS (Phase 3 Exit Criteria)

#### 1. **PII Detection in Guardrails** (Item #5, partial)

**What's Missing**:
- No PII scanning on agent inputs/outputs
- Spec requires: "PII scan: Pass — No PII detected in agent reasoning or tool call parameters"

**Current State**:
- Llama Guard moderation ✅ (checks harmful content)
- PII detection ❌ (missing)

**Options**:
1. **Add Presidio** (Microsoft's open-source PII detector)
   - Effort: 2-3 days
   - Pros: Mature, GDPR-compliant, supports 80+ entity types
   - Cons: Another dependency

2. **Use Llama Guard's PII capabilities** (if available in v3)
   - Effort: 1-2 days
   - Pros: No new dependency
   - Cons: May not be as comprehensive

3. **Defer to Phase 4** (regulated environments only)
   - Effort: 0 days now
   - Pros: Not needed for demos
   - Cons: Can't claim "full governance" in Phase 3

**Recommendation**: **Option 3 (Defer to Phase 4)** unless regulated customer demos are imminent.

---
